MTB CAT1 Peripheral driver library
MISRA-C:2012 Compliance

This page describes MISRA-C:2012 compliance and deviations for the PDL.

MISRA stands for Motor Industry Software Reliability Association. The MISRA specification covers a set of 10 mandatory rules, 110 required rules and 39 advisory rules that apply to firmware design and has been put together by the Automotive Industry to enhance the quality and robustness of the firmware code embedded in automotive devices.

The MISRA specification defines two categories of deviations (see section 5.4 of the MISRA-C:2012 specification):

  • Project Deviations - applicable for all PDL drivers.
  • Specific Deviations - applicable for a single instance in a single file.

Specific deviations are documented in the driver source code, close to the deviation occurrence. For each deviation a macro identifies the relevant rule or directive number, and reason.

Test results may vary across different MISRA checking tools due to implementation differences, compiler settings, and varying interpretations of MISRA guidelines. For instance, MISRA Rule 1.1 may generate false positives when tools enforce only the C99 minimum of 4095 macro identifiers, while modern toolchains (such as GCC, ARM, IAR) typically support significantly higher limits based on available memory and compiler architecture.

◆ delay_bypass

bool cy_stc_tcpwm_motif_hall_sensor_config_t::delay_bypass

will bypass the delay between a Correct Hall Event and the enable of the Multi-Channel Pattern update.

0: Multi-Channel Pattern update is only done after the external SET signal is detected. 1: Multi-Channel Pattern update is done immediately after a Correct Hall Event is detected.

◆ fault_clear

bool cy_stc_tcpwm_motif_hall_sensor_config_t::fault_clear

enable/disable a clear of the Multi-Channel pattern, whenever a Hall Fault Event is detected.

0: Multi-Channel pattern clear with Hall Fault Event disabled. 1: Multi-Channel pattern clear with Hall Fault Event enabled.

◆ debounce_bypass

bool cy_stc_tcpwm_motif_hall_sensor_config_t::debounce_bypass

Setting this true, will disable the usage of the debounce delay pins.

This means that after the low pass filter, the hall inputs are immediately sampled and compared against the expected and current patterns.

◆ pp_compare

bool cy_stc_tcpwm_motif_hall_sensor_config_t::pp_compare

Setting this bitfield to 1, enables the compare logic between the sampled hall input values and the previous hall pattern.

This will be used together with the comparison of the current and expected pattern to generate a wrong hall event.

◆ lut_enable

bool cy_stc_tcpwm_motif_hall_sensor_config_t::lut_enable

Setting this bitfield to 1, will enable the usage of the Hall Look-up table.
